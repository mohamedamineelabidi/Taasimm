# TaaSim `data/` — Runtime vs Study vs Ignored

This folder holds every file the TaaSim pipeline touches at build / run / study
time. Files fall into **three tiers**.

---

## 1. Required for runtime (committed to git)

These must be present for `docker compose up` and the Flink / producer / API
containers to work. They are small and version-controlled.

| File | Used by | Purpose |
|------|---------|---------|
| `zone_mapping.csv` | `producers/config.py`, `flink/jobs/trip_matcher.py` | Casablanca 16-zone catalogue (v3 schema with `adjacent_zones`). |
| `zone_mapping_v4.csv` | `api/main.py`, notebooks `02_*`, `03_*` | Updated v4 zone table (centroid_lat/lon, ae_class). Loaded by API at startup. |
| `h3_zone_lookup.json` | `producers/vehicle_gps_producer.py` (`assign_h3_zone`) | Precomputed H3 → zone lookup for fast GPS snapping. |
| `casablanca_arrondissements.geojson` | Grafana zone overlay, notebooks | Polygon boundaries used by the Geomap dashboard. |
| `casablanca_arrondissements_v4.geojson` | Notebooks (v4 remap study) | v4 zone polygons. |
| `casa_arrondissement_population_2024.csv` | Notebooks, demand baseline | HCP 2024 population per zone, used to weight demand priors. |
| `casablanca_transit.geojson` | Notebooks, future Grafana overlay | OSM-derived bus/tram lines for Casablanca. |
| `casa_synthesis/casa_od_matrix.parquet` | Trip-request producer (`casa_synth` mode) | NYC→Casa OD reweighted matrix (16×16). |
| `casa_synthesis/casa_hourly_demand.parquet` | Trip-request producer | Hourly demand multipliers per zone. |
| `casa_synthesis/gps_trajectory_index.json` | GPS producer (`coupled` mode) | Index of ~500 OSRM polylines by (origin_zone, dest_zone, tier-pair). |
| `casa_synthesis/nyc_to_casa_zone_bridge.csv` | Notebook 12 (Phase-4 build) | NYC TLC → Casa zone crosswalk used during offline synthesis. |

---

## 2. Study artifacts (committed when small, gitignored when large)

Generated by notebooks for documentation / EDA only. The runtime pipeline does
not depend on them.

- `notebooks/*.ipynb` HTML exports → not stored here
- `casablanca_zone_map.html` (interactive folium map) → tracked
- Any `remapped_trips_sample.csv`, `*.preview.csv` → gitignored (see .gitignore)

---

## 3. Ignored raw datasets (kept locally, NOT in git, NOT streamed)

Large third-party datasets used only by offline notebooks and Spark batch ETL
in MinIO. They **must not** be replayed into Kafka directly — Casablanca-native
events go through `casa_synthesis/` → `producers/`.

| Path | Source | Used by |
|------|--------|---------|
| `data/train.csv` | Porto Taxi GPS (1.8 GiB) | Notebook `01_porto_eda.ipynb`, Spark batch ETL via MinIO |
| `data/nyc-tlc/` | NYC TLC trip Parquet (~150 MiB) | Notebook `12_nyc_to_casa_synthesis.ipynb`, Spark ETL |
| `data/hcp-data-casa/` | Morocco HCP demographics | Notebook population analysis |
| `data/opportunity-find/` | Glovo commerce density study | Notebook `15_*` opportunity scoring |
| `data/road_nodes.npy`, `data/road_edges.npy` | OSM road graph build | Notebook `13_*` OSRM routing |
| `data/casa_synthesis/casa_trip_requests.parquet` | Built by notebook 12 (500k trips, 90 days) | Trip-request producer at startup |
| `data/curated_trajectories*.parquet` | Built by notebook 13 from OSRM | Trajectory index builder |

These paths are listed in the top-level `.gitignore`. Re-create them locally
with the corresponding notebook before running `docker compose up`.

---

## Streaming policy (do not violate)

- **Live Kafka events MUST be Casablanca-native** — i.e. derived from
  `casa_synthesis/` artifacts plus producer noise/blackouts.
- Porto raw GPS and NYC TLC Parquet are **offline-only**. They feed notebooks
  and Spark batch jobs; never the producers.
- Cassandra never stores raw precise GPS coordinates — Flink Job 1 snaps lat/lon
  to the zone centroid before persisting.
